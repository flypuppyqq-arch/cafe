## Google Sheets 설정

### 1단계: 새 스프레드시트 만들기

1. 웹브라우저에서 [Google Sheets](https://sheets.google.com) 접속
2. 왼쪽 상단의 **"+ 빈 스프레드시트"** 클릭
3. 스프레드시트 이름을 **"카페_예약_시스템"**으로 변경
   - 상단 왼쪽 "제목 없는 스프레드시트" 클릭하여 변경

### 2단계: 시트 이름 변경

1. 하단의 "시트1" 탭에서 마우스 우클릭
2. **"이름 바꾸기"** 선택
3. 이름을 **"예약목록"**으로 변경

**✅ 완료!** 이제 Google Sheets 설정이 끝났습니다.

## Google Apps Script 설정

### 1단계: Apps Script 열기

1. Google Sheets에서 상단 메뉴 **확장 프로그램** > **Apps Script** 클릭
2. 새 탭에서 Apps Script 에디터가 열립니다

### 2단계: 코드 입력

1. 에디터에 기본으로 있는 코드를 모두 삭제
2. `Code.gs` 파일을 메모장이나 텍스트 에디터로 열기
3. 모든 내용을 복사 (Ctrl+A, Ctrl+C)
4. Apps Script 에디터에 붙여넣기 (Ctrl+V)
5. 상단의 **💾 저장** 아이콘 클릭 (또는 Ctrl+S)

### 3단계: 프로젝트 배포

1. 상단 오른쪽의 **배포** 버튼 클릭
2. **새 배포** 선택
3. 왼쪽의 **⚙️ 설정 아이콘 (톱니바퀴)** 클릭
4. "유형 선택"에서 **웹 앱** 선택
5. 다음 설정 입력:
   - **설명**: "카페 예약 시스템 v1"
   - **다음 계정으로 실행**: "나"
   - **액세스 권한**: "모든 사용자" 선택
6. **배포** 버튼 클릭

### 4단계: 권한 승인

1. "승인 필요" 창이 나타나면 **액세스 승인** 클릭
2. Google 계정 선택
3. "Google에서 이 앱을 확인하지 않았습니다" 경고가 나타나면:
   - **고급** 클릭
   - 하단의 **"[프로젝트명](안전하지 않은 페이지)로 이동"** 클릭
4. **허용** 버튼 클릭

### 5단계: URL 복사

1. 배포 완료 창에서 **웹 앱 URL** 복사
   - 형식: `https://script.google.com/macros/s/...../exec`
2. 이 URL을 메모장에 임시 저장 (매우 중요!)

**✅ 완료!** Google Apps Script 설정이 끝났습니다.

## 웹사이트 파일 설정

### 1단계: script.js 파일 수정

1. `script.js` 파일을 메모장이나 텍스트 에디터로 열기
2. 3번째 줄을 찾으세요:
   ```javascript
   const SCRIPT_URL = 'YOUR_GOOGLE_APPS_SCRIPT_URL_HERE';
   ```
3. 이 줄을 다음과 같이 수정:
   ```javascript
   const SCRIPT_URL = '여기에_복사한_URL_붙여넣기';
   ```
   - 예시:
   ```javascript
   const SCRIPT_URL = 'https://script.google.com/macros/s/AKfycbx.../exec';
   ```
4. 파일 저장 (Ctrl+S)

**⚠️ 주의사항:**
- URL은 작은따옴표 `' '` 안에 넣어야 합니다
- URL 끝이 `/exec`로 끝나는지 확인하세요

### 2단계: 테스트 준비

이제 웹사이트를 열어볼 준비가 되었습니다!

## 웹사이트 배포

### 옵션 1: 로컬에서 테스트 (간단한 방법)

1. `index.html` 파일을 더블클릭하여 웹브라우저로 열기
2. 웹사이트가 정상적으로 보이는지 확인
3. 테스트 예약을 해보세요:
   - 모든 필수 항목 입력
   - "예약하기" 버튼 클릭
   - "예약이 완료되었습니다!" 메시지 확인
4. Google Sheets로 돌아가서 예약이 저장되었는지 확인

### 옵션 2: GitHub Pages로 배포 (무료 호스팅)

#### A. GitHub 계정 만들기

1. [GitHub](https://github.com) 접속
2. **Sign up** 클릭하여 계정 생성

#### B. 새 저장소 만들기

1. GitHub에 로그인 후 오른쪽 상단 **+** > **New repository** 클릭
2. 다음 정보 입력:
   - **Repository name**: `cafe-reservation`
   - **Public** 선택
   - ✅ **Add a README file** 체크
3. **Create repository** 버튼 클릭

#### C. 파일 업로드

1. **Add file** > **Upload files** 클릭
2. 다음 파일들을 드래그하여 업로드:
   - `index.html`
   - `style.css`
   - `script.js`
3. 하단에 커밋 메시지 입력: "카페 예약 시스템 업로드"
4. **Commit changes** 버튼 클릭

#### D. GitHub Pages 활성화

1. 저장소 페이지에서 **Settings** 탭 클릭
2. 왼쪽 메뉴에서 **Pages** 클릭
3. **Source** 섹션:
   - **Branch**: `main` 선택
   - **Folder**: `/ (root)` 선택
4. **Save** 버튼 클릭
5. 1-2분 후 페이지 상단에 URL이 나타납니다:
   - `https://[사용자명].github.io/cafe-reservation/`

**✅ 완료!** 이제 웹사이트가 온라인에 배포되었습니다!

### 옵션 3: 사내 서버에 배포

회사에 웹서버가 있다면:

1. IT 담당자에게 문의
2. 다음 파일들을 웹서버에 업로드:
   - `index.html`
   - `style.css`
   - `script.js`
3. 서버 주소를 공유받아 접속

### 카페 이미지 변경

현재는 임시 이미지가 사용됩니다. 실제 카페 사진으로 변경하려면:

1. 카페 사진을 찍어서 준비 (3장 권장)
2. 이미지를 무료 호스팅 서비스에 업로드:
   - [ImgBB](https://imgbb.com)
   - [Imgur](https://imgur.com)
3. `index.html` 파일에서 이미지 URL 변경:

**수정 위치**: 27, 32, 37번째 줄
```html
<img src="https://via.placeholder.com/400x300/8B4513/FFFFFF?text=카페+내부" alt="카페 내부">
```
→
```html
<img src="업로드한_이미지_URL" alt="카페 내부">
```

### 색상 변경

`style.css` 파일을 수정하여 색상 테마를 변경할 수 있습니다.

**주요 색상 코드** (18, 19번째 줄):
```css
background: linear-gradient(135deg, #8B4513 0%, #D2691E 100%);
```
- `#8B4513`: 진한 갈색 (메인 색상)
- `#D2691E`: 밝은 갈색 (보조 색상)

원하는 색상으로 변경:
- [HTML Color Picker](https://www.w3schools.com/colors/colors_picker.asp) 사용
- 색상 코드를 복사하여 붙여넣기

## 문제 해결

### Q1: 예약 버튼을 눌러도 아무 반응이 없어요

**해결방법:**
1. 웹브라우저의 개발자 도구 열기 (F12 키)
2. "Console" 탭 확인
3. 에러 메시지 확인:
   - "SCRIPT_URL을 먼저 설정해주세요" → `script.js`의 URL 확인
   - "CORS" 에러 → Google Apps Script 배포 설정 확인

### Q2: Google Sheets에 데이터가 저장되지 않아요

**해결방법:**
1. Google Apps Script 페이지로 이동
2. 상단 메뉴 **보기** > **실행 로그** 확인
3. 에러 메시지가 있다면:
   - 권한 문제: 다시 권한 승인
   - 코드 오류: `Code.gs` 코드 재확인

### Q3: "예약이 완료되었습니다" 메시지는 뜨지만 시트에는 없어요

**해결방법:**
1. Google Sheets를 새로고침 (F5)
2. Apps Script의 "실행 로그" 확인
3. 다른 시트에 저장되었는지 확인 (하단 탭들 확인)

### Q4: 웹사이트가 모바일에서 이상하게 보여요

**해결방법:**
- 현재 코드는 반응형으로 설계되어 있습니다
- 웹브라우저 캐시 삭제 후 다시 접속
- 최신 버전의 브라우저 사용

### Q5: GitHub Pages URL이 작동하지 않아요

**해결방법:**
1. Settings > Pages에서 상태 확인
2. "Your site is ready" 메시지가 나올 때까지 대기 (최대 5분)
3. URL을 정확히 복사했는지 확인

### Q6: 과거 날짜로 예약이 가능해요

**해결방법:**
- 현재 코드는 과거 날짜 선택을 방지합니다
- 웹브라우저 캐시 삭제 후 `script.js` 파일 재확인
